# 内网穿透

一个轻量级的 TCP 内网穿透工具，支持多端口映射和自动重连。

## 架构

```
公网服务器（服务端）                 内网机器（客户端）
┌────────────────────┐           ┌─────────────────────┐
│  Control :9000     │◄──────────│ 控制通道（心跳+PING） │
│  Data    :9001     │◄──────────│ 数据通道（隧道）      │
│  :8080  ← 映射端口  │           │  localhost:8003      │
│  :3000  ← 映射端口  │           │  localhost:3000      │
└────────────────────┘           └─────────────────────┘
```

- **控制通道**：客户端与服务器之间维持一条长连接，用于心跳检测和连接通知。
- **数据通道**：每个外部连接创建一个独立的 TCP 隧道，通过服务端转发到客户端。
- **端口映射**：客户端注册 `公网端口:本地端口` 映射，外部流量经服务端透传到内网服务。

## 快速开始

### 1. 服务端（公网服务器）

```bash
# 默认控制端口 9000，数据端口 9001
python3 server.py

# 自定义端口
python3 server.py --control-port 9000 --data-port 9001
```

### 2. 客户端（内网机器）

```bash
# 映射一个端口
python3 client.py <服务器IP> -p 8080:8003

# 映射多个端口
python3 client.py <服务器IP> -p 8080:8003 -p 3000:3000
```

### 示例

将本地 8003 端口的 Web 服务通过服务器 8080 端口暴露到公网：

```bash
# 服务端
python3 server.py

# 客户端
python3 client.py 1.2.3.4 -p 8080:8003
```

访问 `http://1.2.3.4:8080` 即可到达内网机器的 8003 服务。

## 命令行参数

### 服务端

| 参数 | 默认值 | 说明 |
|---|---|---|
| `--control-port` | 9000 | 控制通道端口 |
| `--data-port` | 9001 | 数据通道端口 |
| `--heartbeat-timeout` | 60 | 客户端心跳超时（秒） |
| `--client-listen` | 5 | 控制端口 backlog |

### 客户端

| 参数 | 默认值 | 说明 |
|---|---|---|
| `server` | — | 服务器 IP 地址（必填） |
| `-p` / `--port` | — | 端口映射 `公网端口:本地端口`（可多次指定） |
| `--control-port` | 9000 | 服务端控制端口 |
| `--data-port` | 9001 | 服务端数据端口 |
| `--tunnel-idle-timeout` | 0 | 隧道空闲超时秒数，0 为永不超时 |

## 功能特性

- **多端口映射**：一条连接映射任意多个端口
- **自动重连**：断线后指数退避重连（1s → 2s → 4s → ... → 60s）
- **心跳保活**：TCP keepalive + 应用层 PING/PONG 双保险
- **带宽监控**：实时显示吞吐量
- **空闲超时**：可配置隧道空闲自动关闭
- **线程池**：控制并发线程数，避免资源耗尽
- **Pending 清理**：自动清理超时未响应的连接，防止内存泄漏

## 注意事项

- 服务端需有公网 IP，且防火墙开放控制端口和数据端口
- 数据传输为明文，敏感业务建议配合 SSH 隧道或 TLS 使用
- 服务端不支持多客户端同时连接（适用于一对一场景）
